1.启动

首先先从svn上检索出项目，然后定位到simba-front-end-fay下的fay-admin目录，然后在控制台输入：

npm install

先加载所需要的支持包这个过程可能有点长，或者是直接把已有的文件放到fay-admin的根目录下也行 接下来输入

npm run start

这主要根据我们定义的内容，查看 package.json文件

从返回结果可以看出这是项目的首页，里面的每一块内容都写在单独的文件中。 这里面用了react-route技术也就是react提供的路由技术。 你可能还注意到，Router组件有一个参数history，它的值historyConfig表示，路由的切换由URL的hash变化决定，即URL的#部分发生变化。举例来说，用户访问http://www.example.com/，实际会看到的是http://www.example.com/#/

上面代码中，用户访问/app（比如http://localhost:8000/app）时，加载app组件。因为是现在这个路由组件是嵌套式的，所以第一个Route里面包含其他路由，这样上面代码中，用户访问/app时，会先加载Layout12组件，然后在它的内部再加载Repos组件。

我们可以根据Main.js头部文件的加载知道Layout12组件的位子，然后我们进入该组件

上述代码中{this.props.children}就是用来替换你刚才所访问的app组件。这样你只要访问第一个Route里面的组件都会在这里替换。这里面还包含（头部菜单文件），（左侧菜单组件），也就是说实际上你访问第一个Route内部嵌套的组件时实际是全部刷新页面，只是因为React虚拟Dom的原因，就只有右边内容进行变化，因为React是通过计算虚拟Dom和真是Dom中差别算成最快的变化进行对页面渲染。

我们就用上述经常用到的app组件进行讲解

我们可以看见我们的app/components包底下有一个app.js就是入口文件，以后大家命名组件入口文件时尽量和组件名一致，然后里面其他的的app组件底下的小组件都已App[*].js的形式进行命名，例如app组件里面的表格文件就是AppTable.js这样。

接下来我们来看看App.js里面的代码吧。

/**
 * Created by feichongzheng on 17/1/9.
 */
import React, {Component} from 'react';
import Card from 'antd/lib/card';
import AppSearch from './AppSearch';
import style from '../style/app.css';
import AppTable from './AppTable';

class App extends Component{

    constructor(props) {
        super(props);
        this.getTable = this.getTable.bind(this);
        this.refreshTable = this.refreshTable.bind(this);
    }

    getTable(name){
        this.refs.appTable.params.name = name;
        this.refs.appTable.getData();
    }

    refreshTable(){
        this.refs.appTable.getData();
    }

    render(){
        return (
            <Card>
                <div className={style.appSearch}><AppSearch table={this.getTable} refreshTable={this.refreshTable}/></div>
                <div className={style.appTable}>
                    <AppTable ref="appTable" />
                </div>
            </Card>
        );
    }
}

export default App;

首先这里用到es6中的Class类的知识，这个App类我们继承了React的Component组件，这样我们就能写React的代码了，不过我们首先得继承Component里面的内容，所以我们用了Class的构造器constructor这样在我们这个App类里面的this就会指向本类，而不会指向全局。然后就是在构造器中绑定我们的接下来所要用的方法，其实不绑定也行不过render里面就得改成下面这样：

render(){
    return (
        <Card>
            <div className={style.appSearch}><AppSearch table={this.refreshTable.bind(this)} refreshTable={this.refreshTable.bind(this)}/></div>
            <div className={style.appTable}>
                <AppTable ref="appTable" />
            </div>
        </Card>
    );
}
